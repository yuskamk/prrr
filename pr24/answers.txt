Ответы на вопросы для самоконтроля

1. Какие методы HTTP используются для CRUD операций и в каких случаях?
GET - получение данных (чтение)
POST - создание новых данных
PUT/PATCH - обновление существующих данных
DELETE - удаление данных

2. Как организовать обработку ошибок на фронтенде при работе с API?
- Использование интерцепторов в Axios для централизованной обработки ошибок
- Try-catch блоки в асинхронных функциях
- Состояния для хранения и отображения сообщений об ошибках
- Пользовательские сообщения об ошибках для разных статусов
- Graceful degradation при сбоях сети

3. Что такое CORS и как его настроить для разработки?
CORS (Cross-Origin Resource Sharing) - механизм безопасности браузера, который ограничивает кросс-доменные запросы. Для настройки в разработке:
- Установить пакет cors в Express
- Добавить middleware: app.use(cors())
- Это разрешит запросы с любого домена в development среде

4. Какие подходы к управлению состоянием при интеграции с API вы знаете?
- Локальное состояние в компонентах (useState, useReducer)
- Контекст React (useContext) для глобального состояния
- Сторонние библиотеки (Redux, Zustand, MobX)
- React Query/SWR для управления серверным состоянием
- Комбинация локального и глобального состояния

5. Как реализовать индикацию загрузки без ухудшения UX?
- Отображение скелетонов вместо спиннеров
- Оптимистичные обновления интерфейса
- Пороговое значение задержки для показа индикатора
- Плавные переходы между состояниями
- Прогрессивные индикаторы для длительных операций

6. В чем преимущества и недостатки оптимистичных обновлений?
Преимущества:
- Мгновенная обратная связь для пользователя
- Улучшение воспринимаемой производительности
- Более плавный пользовательский опыт

Недостатки:
- Сложность реализации отката при ошибках
- Возможные конфликты данных
- Усложнение логики приложения
- Риск неконсистентности данных

7. Как организовать валидацию данных на клиенте и сервере?
Клиентская валидация:
- Валидация форм перед отправкой
- Информативные сообщения об ошибках
- Предотвращение невалидных запросов
- Встроенная HTML5 валидация

Серверная валидация:
- Обязательная проверка всех входящих данных
- Валидация схемы данных
- Защита от SQL-инъекций и XSS атак
- Подробные HTTP статусы и сообщения об ошибках

8. Какие методы кэширования данных с сервера можно использовать?
- HTTP кэширование (Cache-Control headers)
- Локальное кэширование в состоянии приложения
- Библиотеки типа React Query для автоматического кэширования
- LocalStorage/SessionStorage для персистентного кэша
- Service Workers для офлайн-работы и кэширования
- Мемоизация результатов запросов
