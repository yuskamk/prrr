## Ответы на вопросы для самоконтроля

### 1. Какие основные проблемы legacy PHP-кода вы выявили?
- Смешение логики и представления
- Уязвимости безопасности (SQL-инъекции, XSS)
- Отсутствие обработки ошибок
- Использование устаревших функций mysql_*
- Плохая структура кода

### 2. В чем преимущества использования фреймворка like Laravel?
- Структурированный подход к разработке
- Встроенные механизмы безопасности
- MVC архитектура для разделения ответственности
- Миграции базы данных для контроля версий схемы
- Встроенная аутентификация и авторизация
- Мощный шаблонизатор Blade
- Система маршрутизации

### 3. Как работает MVC-архитектура в Laravel?
- **Model** (app/Models) - работа с данными, бизнес-логика
- **View** (resources/views) - представление, отображение данных
- **Controller** (app/Http/Controllers) - обработка запросов, посредник между Model и View

### 4. Что такое Composer и для чего он используется?
Composer - менеджер зависимостей для PHP. Используется для:
- Установки пакетов и библиотек
- Автозагрузки классов
- Управления версиями зависимостей
- Создания и установки проектов (composer create-project)

### 5. Какие возможности предоставляет Blade-шаблонизатор?
- Наследование шаблонов (@extends, @section, @yield)
- Циклы и условия (@if, @foreach, @for)
- Включение partials (@include)
- Безопасное экранирование данных ({{ }})
- Директивы (@auth, @guest, @can)
- Компоненты и слоты

### 6. Как организована маршрутизация в Laravel?
Маршруты определяются в файлах routes/*.php:
- web.php - для веб-маршрутов (с сессиями, CSRF защитой)
- api.php - для API маршрутов (с throttle middleware)
- console.php - для команд Artisan
- Параметры маршрутов: {param}, {param?}
- Группы маршрутов с префиксами и middleware

### 7. Какие меры безопасности предоставляет Laravel по сравнению с чистым PHP?
- Защита от SQL-инъекций (Eloquent ORM, Query Builder)
- CSRF защита (токены для форм)
- XSS защита (автоматическое экранирование в Blade)
- Валидация данных (Validator класс)
- Хеширование паролей (bcrypt)
- Middleware для аутентификации и авторизации
- Защита от массового присвоения (fillable/guarded)
