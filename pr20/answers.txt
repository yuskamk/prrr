1. В чем преимущества использования Express.js по сравнению с чистым Node.js для создания серверов?
Преимущества Express.js:
- Упрощенная настройка маршрутизации
- Встроенная поддержка middleware
- Автоматическая обработка статических файлов
- Упрощенная работа с cookies и сессиями
- Большое количество готовых middleware
- Активное сообщество и хорошая документация
- Меньше boilerplate кода
- Быстрая разработка API

2. Что такое middleware в Express.js и каков порядок их выполнения?
Middleware - это функции, которые имеют доступ к объектам запроса (req), ответа (res) и следующей функции middleware. Они выполняют различные задачи: логирование, аутентификация, парсинг данных и т.д.
Порядок выполнения:
1. Middleware выполняются в порядке их объявления в коде
2. Каждый middleware должен вызвать next() для передачи управления следующему
3. Если middleware не вызывает next(), цепочка прерывается
4. Ошибки передаются через next(error)

3. Как организовать модульную структуру роутеров в Express приложении?
Организация модульной структуры:
1. Создаем отдельные файлы для каждого роутера (users.js, products.js)
2. Используем express.Router() для создания экземпляра роутера
3. Определяем маршруты на роутере
4. Экспортируем роутер через module.exports
5. В основном файле подключаем через app.use('/path', router)
6. Группируем связанные маршруты вместе

4. Какие методы роутинга поддерживает Express.js и для чего они используются?
Основные методы:
- GET - получение данных
- POST - создание новых данных
- PUT - полное обновление данных
- PATCH - частичное обновление данных
- DELETE - удаление данных
- ALL - обработка всех HTTP методов для определенного пути

5. Как обрабатывать параметры URL и query-параметры в Express?
Параметры URL:
- Определяются в пути: /users/:id
- Доступны через: req.params.id
Query-параметры:
- Передаются в URL: /products?category=electronics&page=1
- Доступны через: req.query.category, req.query.page

6. В чем разница между app.use() и app.METHOD()?
app.use():
- Используется для middleware и роутеров
- Обрабатывает все HTTP методы
- Может быть применен ко всем маршрутам или определенному пути
app.METHOD():
- Обрабатывает только конкретный HTTP метод (GET, POST, PUT, DELETE)
- Используется для определения конечных точек API
- Более специфичен чем app.use()

7. Как организовать централизованную обработку ошибок в Express приложении?
Централизованная обработка ошибок:
1. Создаем middleware с 4 параметрами: (err, req, res, next)
2. Помещаем его в конец цепочки middleware
3. Передаем ошибки через next(error) из других middleware
4. Обрабатываем разные типы ошибок соответствующим образом

8. Что такое router.param() и для чего он используется?
router.param() - специальный middleware, который выполняется когда в маршруте встречается определенный параметр.
Используется для:
- Валидации параметров маршрута
- Поиска данных в базе данных по параметру
- Добавления найденных данных в объект запроса
- Предварительной обработки параметров
